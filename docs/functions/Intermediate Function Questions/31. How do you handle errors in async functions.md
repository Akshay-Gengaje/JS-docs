# How do you handle errors in async functions?

Handling errors properly in `async` functions is **crucial** for building robust applications. Since `async` functions return Promises, errors can be managed using **`try...catch` blocks**, `.catch()`, or **global error handlers**.

## **ðŸ”¹ 1. Using `try...catch` (Best Practice âœ…)**
ðŸ“Œ The most common way to handle errors in `async` functions is using `try...catch`.  
ðŸ“Œ This prevents the app from crashing and provides a **clean error handling mechanism**.

### **âœ” Example: Handling API Errors**
```javascript
async function fetchData() {
    try {
        let response = await fetch("https://jsonplaceholder.typicode.com/posts/1");

        // Check if the response is OK
        if (!response.ok) {
            throw new Error(`HTTP error! Status: ${response.status}`);
        }

        let data = await response.json();
        console.log("Data received:", data);
    } catch (error) {
        console.log("Error fetching data:", error.message);
    }
}

fetchData();
```
âœ… **Why use `try...catch`?**  
- **Catches network errors, JSON parsing errors, and other exceptions.**  
- **Prevents unhandled Promise rejections.**  
- **Keeps code clean and readable.**

## **ðŸ”¹ 2. Using `.catch()` on the Returned Promise**
ðŸ“Œ Since an `async` function **returns a Promise**, errors can also be caught using `.catch()`.  

### **âœ” Example: `.catch()` Instead of `try...catch`**
```javascript

    let response = await fetch("invalid-url"); // This will fail
    let data = await response.json();
    return data;
}

// Handle error using .catch()
fetchData().then(console.log).catch(error => {
    console.log("Caught an error:", error.message);
});
```
âœ… **Pros of `.catch()`:**  
- **Useful when calling async functions from non-async functions.**  
- **Avoids the need for explicit `try...catch` inside the function.**  

âŒ **Cons:**  
- **Cannot handle synchronous errors inside the function.**  
- **Error handling is done outside the function, which may lead to missing errors if `.catch()` is forgotten.**

## **ðŸ”¹ 3. Using `window.onerror` or `process.on('unhandledRejection')` (Global Handling)**
ðŸ“Œ **Best for catching unhandled async errors across the entire application.**  

### **âœ” Example: Handling Unhandled Promise Rejections**
For **browsers**:
```javascript

    console.error("Unhandled rejection caught:", event.reason);
});
```

For **Node.js**:
```javascript

    console.error("Unhandled rejection at:", promise, "reason:", reason);
});
```
âœ… **Why use global error handling?**  
- **Catches errors in async functions if `.catch()` is forgotten.**  
- **Prevents crashes due to unhandled Promise rejections.**  

âŒ **Drawback:**  
- **Not specific to a function, so debugging might be harder.**

## **ðŸ”¹ 4. Using `finally()` for Cleanup**
ðŸ“Œ `finally()` runs **whether the Promise succeeds or fails**, making it useful for cleanup tasks.  

### **âœ” Example: Cleanup After an Async Operation**
```javascript

    try {
        let response = await fetch("https://jsonplaceholder.typicode.com/posts/1");
        let data = await response.json();
        console.log("Data received:", data);
    } catch (error) {
        console.log("Error fetching data:", error.message);
    } finally {
        console.log("Cleanup: Closing connection...");
    }
}

fetchData();
```
âœ… **Why use `finally()`?**  
- **Ensures cleanup happens, regardless of success or failure.**  
- **Useful for closing database connections, stopping loaders, etc.**

## **ðŸ”¹ 5. Handling Multiple Async Errors with `Promise.all()`**
ðŸ“Œ When using `Promise.all()`, if **one Promise fails, the entire operation fails**.  

### **âœ” Example: Handling Errors in `Promise.all()`**

```javascript
            fetch("https://jsonplaceholder.typicode.com/users/1").then(res => res.json()),
            fetch("https://jsonplaceholder.typicode.com/posts/1").then(res => res.json())
        ]);
        console.log("User:", user);
        console.log("Posts:", posts);
    } catch (error) {
        console.log("Error fetching data:", error.message);
    }
}

fetchMultipleData();
```
âœ… **How does this help?**  
- **Catches any failed request inside `Promise.all()`.**  
- **Ensures both requests run in parallel.**
