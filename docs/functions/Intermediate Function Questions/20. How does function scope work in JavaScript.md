# How does function scope work in JavaScript?

Function scope in JavaScript refers to the **visibility and accessibility of variables** declared inside a function. A variable declared inside a function **is only accessible within that function** and is **not available outside** of it.

### **‚úî Key Points:**
1. **Variables declared inside a function (`var`, `let`, `const`) are local to that function.**
2. **Outer code cannot access function-scoped variables.**
3. **Functions can access variables from outer (parent) scopes (lexical scoping).**

---

## **üîπ Example 1: Basic Function Scope**
```javascript
function example() {
    let message = "Hello, World!";
    console.log(message); // ‚úÖ Accessible inside the function
}

example();
console.log(message); // ‚ùå ReferenceError: message is not defined
```
‚úî The variable `message` is **declared inside `example()`**, so it **only exists within that function**.

---

## **üîπ Example 2: Function Scope with `var` vs. `let`/`const`**
### **Using `var` (Function Scoped)**
```javascript
function testVar() {
    if (true) {
        var x = 10;
    }
    console.log(x); // ‚úÖ Works (var is function-scoped)
}
testVar();
```
‚úî `var` is **function-scoped**, so `x` is **accessible anywhere inside `testVar()`**, even outside the block `{}`.

### **Using `let` and `const` (Block Scoped)**
```javascript
function testLetConst() {
    if (true) {
        let y = 20;
        const z = 30;
    }
    console.log(y); // ‚ùå ReferenceError
    console.log(z); // ‚ùå ReferenceError
}
testLetConst();
```
‚úî `let` and `const` are **block-scoped**, meaning they **only exist within the `{}` block**.

---

## **üîπ Example 3: Function Scope and Nested Functions**
### **Lexical (Static) Scoping**
A function **can access variables from its parent function**, but the parent **cannot access child function variables**.

```javascript
function outerFunction() {
    let outerVar = "I am outer";

    function innerFunction() {
        let innerVar = "I am inner";
        console.log(outerVar); // ‚úÖ Accessible (lexical scope)
        console.log(innerVar); // ‚úÖ Accessible (inside its own function)
    }

    innerFunction();
    console.log(innerVar); // ‚ùå ReferenceError (innerVar is scoped to innerFunction)
}

outerFunction();
```
‚úî `innerFunction()` **can access `outerVar`**, but `outerFunction()` **cannot access `innerVar`**.

---

## **üîπ Example 4: Function Scope vs. Global Scope**
```javascript
let globalVar = "I am global";

function demoScope() {
    let localVar = "I am local";
    console.log(globalVar); // ‚úÖ Accessible (global scope)
    console.log(localVar);  // ‚úÖ Accessible (local scope)
}

demoScope();
console.log(localVar); // ‚ùå ReferenceError (localVar is function-scoped)
```
‚úî **Global variables are accessible inside functions**, but **local variables are not accessible globally**.

---

## **üîπ Example 5: Function Parameters Are Function Scoped**
```javascript
function add(a, b) {
    console.log(a + b); // ‚úÖ Parameters are function-scoped
}

add(5, 10);
console.log(a); // ‚ùå ReferenceError (a is not defined outside `add()`)
```
‚úî Function parameters (`a` and `b`) **only exist inside the function**.

---

## **üîπ Summary**
| Scope Type | Accessible Inside Function? | Accessible Outside Function? |
|------------|---------------------------|----------------------------|
| **Function Scope (`var`, `let`, `const`)** | ‚úÖ Yes | ‚ùå No |
| **Block Scope (`let`, `const`)** | ‚úÖ Only inside the block `{}` | ‚ùå No |
| **Global Scope (`var`, `let`, `const` outside function)** | ‚úÖ Yes | ‚úÖ Yes |

---

## **üîπ Key Takeaways**
‚úÖ **Function scope restricts variable access to within the function.**  
‚úÖ **Lexical scoping allows inner functions to access outer variables.**  
‚úÖ **`var` is function-scoped, while `let` and `const` are block-scoped.**  
‚úÖ **Function parameters are function-scoped.**
