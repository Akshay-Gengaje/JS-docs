# What is the difference between prototype methods and function methods?

JavaScript allows methods to be added to **functions** and **prototypes**, but they serve different purposes. Let‚Äôs break it down:  

---

## **üîπ 1Ô∏è‚É£ What Are Prototype Methods?**  
üìå **Prototype methods** are functions defined on a **constructor function‚Äôs prototype**, making them **available to all instances** of that constructor.  
üìå These methods are **shared across all instances**, improving **memory efficiency**.  

### **‚úÖ Example: Defining a Prototype Method**
```javascript
function Person(name) {
    this.name = name;
}

// Adding a method to the prototype
Person.prototype.greet = function() {
    return `Hello, my name is ${this.name}`;
};

const alice = new Person("Alice");
console.log(alice.greet()); // Output: Hello, my name is Alice

const bob = new Person("Bob");
console.log(bob.greet()); // Output: Hello, my name is Bob
```
### **‚úî Key Takeaways**
- **Prototype methods are inherited** by all instances of `Person`.  
- **Memory-efficient**: The method is stored in `Person.prototype`, not duplicated for each instance.  
- Used in **class-based programming and object inheritance**.  

---

## **üîπ 2Ô∏è‚É£ What Are Function Methods?**  
üìå **Function methods** are methods available on JavaScript's built-in `Function` object.  
üìå These methods **only apply to functions** and are used for **manipulating functions**.  

### **‚úÖ Example: Function Methods**
```javascript
function sayHello() {
    console.log("Hello, World!");
}

// Using the .call() function method
sayHello.call(); // Output: Hello, World!

// Using the .bind() function method
const greet = sayHello.bind();
greet(); // Output: Hello, World!
```
### **‚úî Key Takeaways**
- Function methods like `.call()`, `.apply()`, `.bind()`, `.toString()` work **only on functions**.  
- They **modify function execution**, change `this` context, or convert functions to strings.  
- Used in **higher-order functions, event handling, and functional programming**.  

---

## **üîπ 3Ô∏è‚É£ Key Differences Between Prototype Methods and Function Methods**  

| Feature | **Prototype Methods** | **Function Methods** |
|---------|----------------------|----------------------|
| **Where It Exists** | On an object‚Äôs **prototype** | On the built-in `Function` object |
| **Usage** | Used to define methods for **objects & instances** | Used to manipulate **functions** |
| **Scope** | Applies to all **instances** of a constructor | Applies only to **functions** |
| **Memory Efficiency** | **Efficient** (methods are shared across instances) | Not applicable (used for function behavior) |
| **Examples** | `Person.prototype.greet()` | `function.call()`, `function.bind()`, `function.apply()` |

---

## **üîπ 4Ô∏è‚É£ When to Use Each?**
‚úÖ **Use Prototype Methods When:**  
‚úî You are defining **methods for object instances** (e.g., `Person.prototype.greet`).  
‚úî You want methods to be **shared** across instances for **memory efficiency**.  
‚úî You are working with **object-oriented programming (OOP)**.  

‚úÖ **Use Function Methods When:**  
‚úî You need to **modify function execution** (e.g., `.call()`, `.bind()`).  
‚úî You want to **change the `this` context** dynamically.  
‚úî You are working with **functional programming concepts**.  

---

## **üîπ Conclusion**
‚úî **Prototype methods** are used to **add methods to objects** so they are shared across all instances.  
‚úî **Function methods** belong to the `Function` object and help in **modifying function behavior**.  
‚úî Understanding both concepts is crucial for **object-oriented and functional programming in JavaScript**.
