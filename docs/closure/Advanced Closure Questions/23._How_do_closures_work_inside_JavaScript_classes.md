# How do closures work inside JavaScript classes?

Closures in JavaScript classes help **retain access** to private variables and methods, allowing **encapsulation** and controlled access to internal data.  

---

## **1. Closures in Class Methods**
### **Example: Class Without Closures**
```javascript
class Counter {
    constructor() {
        this.count = 0;
    }

    increment() {
        this.count++;
        console.log(`Count: ${this.count}`);
    }
}

const counter = new Counter();
counter.increment(); // Count: 1
counter.increment(); // Count: 2
console.log(counter.count); // â— Publicly accessible: 2
```
âŒ **Problem:** `count` is public. Anyone can modify it.

## **2. Using Closures for Private Variables**
We can use closures to **restrict direct access** to variables.

### âœ… **Example: Private Variables Using Closures**
```javascript
class Counter {
    constructor() {
        let count = 0; // Private variable (closure)

        this.increment = function () {
            count++;
            console.log(`Count: ${count}`);
        };

        this.getCount = function () {
            return count;
        };
    }
}

const counter = new Counter();
counter.increment(); // Count: 1
counter.increment(); // Count: 2
console.log(counter.getCount()); // âœ… 2
console.log(counter.count); // âŒ Undefined (Encapsulated)
```
âœ”ï¸ **Closure retains `count`**, making it private.  
âœ”ï¸ **Only `increment()` and `getCount()` can access `count`.**

## **3. Closures in Class Methods with `setTimeout`**
When using `setTimeout` inside a class method, closures help preserve `this`.

### âŒ **Problem: Losing `this` in a Callback**

    start() {
        console.log("Starting...");
        setTimeout(function () {
            console.log("Timer finished!");
        }, 2000);
    }
}

```

            console.log("Timer finished!");
        }, 2000);
    }
}
```
âœ”ï¸ Arrow functions **inherit `this` from the enclosing scope**, fixing the issue.

        }, 2000);
    }
}
```
âœ”ï¸ Storing `this` as `self` helps maintain context inside `setTimeout()`.

## **4. Closures for Private Methods in Classes**
We can **hide methods** using closures.

### âœ… **Example: Private Method**

        };

            if (amount > 0) {
                balance += amount;
                logTransaction(amount);
            }
        };

        this.getBalance = function () {
            return balance;
        };
    }
}

console.log(account.getBalance()); // âœ… 150
console.log(account.balance); // âŒ Undefined (private)

```
âœ”ï¸ **Encapsulated `balance` and `logTransaction`** using closures.  
âœ”ï¸ **Only `deposit()` and `getBalance()` can access them.**
