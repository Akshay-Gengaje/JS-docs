# How can closures be used to implement a module pattern?

Closures are a fundamental part of the **Module Pattern** in JavaScript. This pattern is used to **encapsulate private data** and expose only selected functions or variables, creating a controlled public API.  

---

## **1. What is the Module Pattern?**  
The **Module Pattern** is a design pattern that:  
‚úîÔ∏è **Encapsulates private variables and functions** using closures.  
‚úîÔ∏è **Exposes only necessary methods** through a returned object.  
‚úîÔ∏è **Prevents pollution** of the global scope.  

üîπ It helps organize code, **prevent naming conflicts**, and **create reusable components**.  

---

## **2. Implementing the Module Pattern with Closures**
### ‚úÖ **Example: Simple Module**
```javascript
const CounterModule = (function () {
    let count = 0; // Private variable (closure)

    function increment() {
        count++;
        console.log(`Count: ${count}`);
    }

    function decrement() {
        count--;
        console.log(`Count: ${count}`);
    }

    function getCount() {
        return count;
    }

    return {
        increment,
        decrement,
        getCount
    };
})();

CounterModule.increment(); // ‚úÖ "Count: 1"
CounterModule.increment(); // ‚úÖ "Count: 2"
console.log(CounterModule.getCount()); // ‚úÖ 2
console.log(CounterModule.count); // ‚ùå Undefined (Private)
```
‚úîÔ∏è **Encapsulates `count` inside a closure** (cannot be accessed directly).  
‚úîÔ∏è **Only `increment()`, `decrement()`, and `getCount()` are exposed**.

## **3. Advanced Module with Private Methods**
We can extend the pattern by adding **private helper functions**.

### ‚úÖ **Example: Secure Bank Module**
```javascript
const BankModule = (function () {
    let balance = 1000; // Private variable

    function logTransaction(amount, type) { // Private function
        console.log(`Transaction: ${type} $${amount}`);
    }

    return {
        deposit(amount) {
            balance += amount;
            logTransaction(amount, "Deposit");
        },
        withdraw(amount) {
            if (amount <= balance) {
                balance -= amount;
                logTransaction(amount, "Withdraw");
            } else {
                console.log("Insufficient funds!");
            }
        },
        getBalance() {
            return balance;
        }
    };
})();

BankModule.deposit(200); // ‚úÖ "Transaction: Deposit $200"
BankModule.withdraw(100); // ‚úÖ "Transaction: Withdraw $100"
console.log(BankModule.getBalance()); // ‚úÖ 1100
console.log(BankModule.balance); // ‚ùå Undefined (Private)
console.log(BankModule.logTransaction); // ‚ùå Undefined (Private)
```
‚úîÔ∏è **Encapsulates `balance` and `logTransaction()`**.  
‚úîÔ∏è **Only public methods can access private data**.

## **4. The Module Pattern with ES6 `const` and `let`**
With ES6, we can define modules using `const` and `let` inside an **IIFE (Immediately Invoked Function Expression)**.

### ‚úÖ **Example: ES6 Module**
```javascript
const UserModule = (function () {
    let users = []; // Private array

    return {
        addUser(name) {
            users.push(name);
            console.log(`${name} added.`);
        },
        getUsers() {
            return [...users]; // Returning a copy
        }
    };
})();

UserModule.addUser("Alice"); // ‚úÖ "Alice added."
UserModule.addUser("Bob"); // ‚úÖ "Bob added."
console.log(UserModule.getUsers()); // ‚úÖ ["Alice", "Bob"]
console.log(UserModule.users); // ‚ùå Undefined (Private)
```
‚úîÔ∏è **Encapsulates `users` array**.  
‚úîÔ∏è **Protects data by returning a copy**.
